render​Content() 

open func renderContent() -> RenderObject?

Should be used by inheriting Widgets to create their rendered representation. This function is called internally by the render() (which then calls updateRenderState()) function. So don't call this directly. It is allowed to access the current render state, modify the current RenderObject and return it again (this approach might increase performance for Widgets that often need to rerender as it avoids some class instatiations and mounting in the RenderObjectTree).
